# Device Integration: Philips Hue Lighting

## Status

Protocol documented and stable. Control is via the local Hue Bridge REST API. No cloud dependency. The existing play-triggered dimming (via an *arr app or webhook) stays in place; this adapter provides manual scene and level control in the unified UI.

## Transport

- Protocol: HTTPS to the local Hue Bridge.
- Authentication: an application key ("username") generated by pressing the bridge link button and issuing a POST. Store this key in the `HUE_APP_KEY` environment variable.
- Two API generations are available:
  1. Legacy v1: `https://<bridge-ip>/api/<app-key>/lights/<id>/state`, `PUT` with a JSON body such as `{"on": true, "bri": 254}`. Simple and sufficient for phase 1.
  2. CLIP v2: `https://<bridge-ip>/clip/v2/resource/...` with the `hue-application-key` header. Preferred for scenes and grouped rooms if richer control is wanted.

## Light and Scene State (v1 example)

| Action | Method | Body |
|---|---|---|
| Turn on | `PUT .../lights/<id>/state` | `{"on": true}` |
| Turn off | `PUT .../lights/<id>/state` | `{"on": false}` |
| Set brightness | `PUT .../lights/<id>/state` | `{"on": true, "bri": 128}` |
| Set color | `PUT .../lights/<id>/state` | `{"on": true, "hue": 10000, "sat": 254, "bri": 254}` |

`bri` and `sat` range 0–254; `hue` ranges 0–65535.

## Required Capabilities (Phase 1)

| Capability | Purpose |
|---|---|
| Recall scene | One-tap scene buttons (for example, Movie, Bright, Off) |
| Set group level | Master brightness for the theater room/group |
| Toggle group on/off | Quick all-on / all-off |
| Read group state | Reflect current on/off and level in the UI |

## Adapter Requirements

1. Configure the bridge IP and target room/group; scenes are referenced by scene identifier (v1) or scene resource (v2).
2. Provide a small set of named scene buttons defined in `config/devices.yaml`.
3. Do not duplicate or override the existing play-triggered dimming automation. The app offers manual override; the *arr/webhook automation remains the play-driven trigger.

## Configuration

```yaml
hue:
  bridge_ip: "192.168.x.x"
  # app key provided via HUE_APP_KEY environment variable
  room_group_id: "1"
  scenes:
    movie: "<scene-id>"
    bright: "<scene-id>"
    off: "<scene-id>"
```
